## Study Techniques (Science-Backed)

### ✅ What Works

1. **Spaced Repetition:** Review same topic multiple times over weeks (not cramming)
2. **Active Recall:** Answer questions from memory before looking at answer
3. **Elaboration:** Explain concept to someone else or write your own explanation
4. **Interleaving:** Mix different topics in single study session (not blocked)
5. **Retrieval Practice:** Test yourself constantly (quizzes, mock interviews)
6. **Concrete Examples:** Code examples > abstract explanations

### ❌ What Doesn't Work

1. **Passive reading:** Just reading without coding
2. **Highlighting:** Marking textbook doesn't transfer to memory
3. **Massed practice:** Cramming same topic for 3 hours straight
4. **Comfort zone:** Studying only what you already know
5. **No mistakes:** Not attempting hard questions before studying answer
6. **Waiting for feeling of readiness:** You'll never feel 100% ready; just practice

### Your Study Approach (Optimized)

**Monday:** Learn Topic X (read + code)
**Tuesday:** Practice Topic X (5 questions, mini-project)
**Wednesday:** Learn Topic Y (read + code)
**Thursday:** Practice Topics X + Y mixed (interleaving)
**Friday:** Full mock interview on X + Y
**Weekend:** Review weak areas from Friday, study Topic Z

This follows spaced repetition + interleaving + active recall.

## Common Pitfalls & How to Avoid Them

| Pitfall | Why It Happens | Fix |
|---------|---|---|
| Learn topics in isolation | Easy to compartmentalize | Build mini-projects connecting topics |
| Memorize answers | Faster short-term | Practice variations; focus on understanding |
| Skip multithreading prep | It's hard | Do it first when fresh; spend double time here |
| Don't code examples | Feels efficient | Code every single example; you'll learn 5x faster |
| Study only weak topics | Comfortable | Interleave; refresh strong topics too |
| Don't do mock interviews | Too intimidating | Start with 2 questions; build to 10 |
| Cram final week | Time efficient? | Space out; cram doesn't work for interviews |
| Don't understand GC | "Not important" | Asked in 60% of interviews; can't skip |
| Skip design patterns | Too many | Focus on 5: Singleton, Factory, Builder, Observer, Strategy |
