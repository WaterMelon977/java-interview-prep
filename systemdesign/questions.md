# 🔥 TIER 1 — VERY HIGH FREQUENCY (Almost every interview)

## 🧠 Fundamentals + Thinking Ability

* What is system design?
* What is the difference between High-Level Design and Low-Level Design?
* What are functional vs non-functional requirements?
* What are scalability, availability, and reliability?
* How do you approach a system design problem?
* How do you handle trade-offs in system design?
* When would you choose microservices over monolith?
* When is a monolith a better choice?
* What are the drawbacks of microservices?

---

## 🌐 API Design (EXTREMELY COMMON)

* How do you design a REST API?
* What are REST principles?
* What is idempotency?
* Which HTTP methods are idempotent?
* What is the difference between PUT and POST?
* How do you design APIs for large-scale systems?
* How do you handle versioning in APIs?
* What are common HTTP status codes and when to use them?

---

## 💾 Database Design (CRITICAL AREA)

* How do you design a database schema for a given system?
* When would you use SQL vs NoSQL?
* How do you model relationships between entities?
* How do you design tables for users, orders, and payments?
* What is normalization and when would you denormalize?
* How do indexes improve performance?
* When should you avoid indexing?
* How do you handle transactions in distributed systems?
* What are ACID properties?

---

## 🚀 Scalability + Performance (VERY IMPORTANT)

* What is vertical vs horizontal scaling?
* How do you handle high traffic in a system?
* What is load balancing and why is it needed?
* How do you scale a backend system?
* What are bottlenecks in a system?
* How do you reduce latency?

---

## ⚙️ Caching (HIGH FREQUENCY)

* What is caching and why is it used?
* When should you use caching?
* What are different types of caching?
* What is cache invalidation?
* What are cache consistency challenges?
* Where would you place cache in a system?

---

# ⚡ TIER 2 — HIGH FREQUENCY (Asked often)

## 🔄 Asynchronous Processing

* What is asynchronous processing?
* When would you use async over sync?
* What are the benefits of message queues?
* How do message queues improve system performance?
* What is eventual consistency?
* What is the difference between Kafka and RabbitMQ?

---

## ⚠️ Fault Tolerance & Reliability

* What happens if a service fails?
* How do you design a fault-tolerant system?
* What is a circuit breaker?
* Why do we need retries and timeouts?
* How do you prevent cascading failures?
* How do you handle partial failures in distributed systems?

---

## 🔐 Security

* How do you secure APIs?
* What is authentication vs authorization?
* What is JWT and how does it work?
* How do you handle secure communication between services?
* What are common API security best practices?

---

## 📊 Logging & Monitoring

* How do you debug production issues?
* What is centralized logging?
* What is monitoring and why is it important?
* What metrics would you track in a system?
* How do you trace a request across services?

---

# 🚀 TIER 3 — MEDIUM FREQUENCY (Good companies / better rounds)

## 🐳 Deployment & DevOps Basics

* What is Docker?
* What is the difference between container and VM?
* How do you deploy a Spring Boot application?
* What is Kubernetes and why is it used?
* What is a pod in Kubernetes?
* How do you scale services in production?

---

## 🔁 Rate Limiting & Protection

* What is rate limiting?
* Why is rate limiting important?
* How do you implement rate limiting?
* What is throttling?
* How do you protect APIs from abuse?

---

## 🧩 Design Problems (VERY IMPORTANT PRACTICE AREA)

### General Design Questions

* How would you design a URL shortener?
* How would you design an e-commerce system?
* How would you design a notification system?
* How would you design a file upload service?
* How would you design a basic chat system?

---

### Deep Follow-ups (Very Common)

* What APIs would you design for this system?
* What database schema would you use?
* How would you scale this system?
* Where would you use caching?
* How would you handle failures?
* How would you handle high traffic spikes?

Most candidates fail system design because:

* They jump into solution without clarifying requirements ❌
* They don’t structure answers ❌
* They ignore trade-offs ❌

If you can:

* Think clearly
* Explain step-by-step
* Justify decisions

👉 You’ll outperform most candidates at your level.
